From Raw Columns to Perfect Schema Alignment

The Schema Mapping engine processes each schema-dataset pair with a sequential AI pipeline, combining three independent scoring signals into a unified confidence score for every column match.

  • String similarity via fuzzy matching handles straightforward exact-match columns like transaction_date
  • Semantic vector embeddings capture conceptual similarity between differently-named but equivalent fields
  • LLM contextual reasoning understands business domain nuance — abbreviations, acronyms, alias naming
  • Mappings above 0.75 auto-approved instantly; between 0.50–0.75 flagged for human review
  • Unmapped columns clearly identified — e.g., tpm_id — full transparency, no hidden failures
  • position_cfg.json: all 23 columns mapped with confidence 1.00 — zero unmapped
